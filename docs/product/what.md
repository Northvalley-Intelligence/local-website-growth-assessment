# What

The product accepts a public website URL, performs a safe same-domain assessment, extracts public signals, scores the website against local lead-generation categories, and produces a business-friendly report.

It does not perform vulnerability testing, login testing, form submission, competitor surveillance, or private analytics analysis.

Submission is asynchronous. A submitted URL creates trackable assessment work immediately, then crawl and scoring execution happen separately. The user sees truthful status until the report is complete.

Report generation requires evidence sufficiency. If the system cannot crawl meaningful public pages or cannot extract enough readable public content, it shows an insufficient-evidence status instead of a normal scored report.

Scores are explainable. Each category score shows the evidence found, evidence missing, how the evidence contributed to the score, why it matters, and how to improve it.

Score explanations are traceable. Category summaries show the checks that passed and the checks that were missing, then detailed findings explain why each item matters in business language.

Missing signals explain existing-content distinctions. For example, an owner should be able to understand that a biography page may help human visitors while LocalBusiness schema provides a separate machine-readable signal for search engines and AI systems.

Reports include evidence confidence. Successful, partial, failed, and insufficient-evidence outcomes must be visibly different.

The report is organized for business decisions. It starts with the executive summary, biggest opportunity, and top recommended actions before score details and diagnostics. Detailed findings should support the recommendation, not bury it.
